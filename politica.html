Política de Privacidade

Última atualização: Janeiro de 2025

📋Introdução

A presente Política de Privacidade descreve como a aplicação FaturaApp recolhe, utiliza e protege os dados dos seus utilizadores, em conformidade com o Regulamento Geral sobre a Proteção de Dados (RGPD — Regulamento UE 2016/679) e a legislação portuguesa aplicável.

Ao utilizar a FaturaApp, o utilizador aceita as práticas descritas nesta política. Recomendamos a leitura integral antes de utilizar a aplicação.

O responsável pelo tratamento de dados é o titular da conta de negócio que utiliza a aplicação para gestão das suas faturas.

📊Dados recolhidos

A FaturaApp recolhe e processa os seguintes dados para o correto funcionamento:

  • Dados de faturas: número de fatura, fornecedor, NIF do fornecedor, NIF do cliente, data, valor total, IVA, ATCUD e categoria
  • Dados de câmara: acesso temporário à câmara do dispositivo para leitura de QR Code de faturas AT — as imagens não são armazenadas
  • Dados de conta Google: quando utilizada a integração com Google Sheets, são utilizadas as credenciais de acesso à conta Google do utilizador
  • Dados de configuração: nome do negócio, NIF da empresa, email do contabilista e preferências da aplicação

A aplicação não recolhe dados de localização, dados biométricos, contactos do dispositivo nem informações de navegação.

🎯Finalidade do tratamento

Os dados recolhidos são utilizados exclusivamente para:

  • Registo e organização de faturas da empresa
  • Leitura automática de QR Code de faturas emitidas pela Autoridade Tributária portuguesa
  • Sincronização com Google Sheets para arquivo e partilha com o contabilista
  • Geração de relatórios e exportação em formato CSV
  • Cálculo de totais e IVA dedutível para efeitos contabilísticos

Os dados não são utilizados para fins de marketing, publicidade, criação de perfis ou partilha com terceiros não identificados nesta política.

💾Armazenamento de dados

Os dados das faturas são armazenados da seguinte forma:

  • Localmente no dispositivo: os dados são guardados na memória local (localStorage) do dispositivo, sem envio automático para servidores externos
  • Google Sheets (opcional): quando o utilizador configura a integração, os dados são sincronizados com a folha de cálculo Google do utilizador, ficando sujeitos à Política de Privacidade da Google
  • Google Apps Script: o backend da aplicação corre na infraestrutura Google, utilizando as credenciais e permissões configuradas pelo utilizador
Os dados armazenados localmente permanecem no dispositivo até o utilizador os eliminar ou desinstalar a aplicação. Não existe servidor próprio da FaturaApp que armazene dados dos utilizadores.

📷Permissões da aplicação

A FaturaApp solicita as seguintes permissões no dispositivo Android:

  • Câmara: necessária para leitura de QR Code de faturas AT. O acesso é ativado apenas quando o utilizador abre a função de digitalização e é terminado imediatamente após a leitura. Nenhuma imagem é guardada.
  • Internet: necessária para sincronização com Google Sheets e Google Apps Script quando configurado pelo utilizador.

Todas as permissões são solicitadas de forma transparente e podem ser revogadas nas definições do dispositivo a qualquer momento.

🤝Partilha de dados com terceiros

A FaturaApp partilha dados apenas com os seguintes terceiros, sempre por iniciativa e configuração do próprio utilizador:

  • Google LLC: quando o utilizador configura a integração com Google Sheets ou Google Apps Script. Sujeito à Política de Privacidade da Google.
  • Contabilista do utilizador: quando o utilizador utiliza a função de envio para contabilidade, os dados das faturas são enviados para o endereço de email configurado pelo próprio utilizador.

A FaturaApp não vende, cede nem partilha dados com anunciantes, corretores de dados ou quaisquer outras entidades terceiras.

🔒Segurança dos dados

A proteção dos dados é assegurada através das seguintes medidas:

  • Os dados armazenados localmente estão protegidos pelo sistema de segurança do dispositivo Android
  • A comunicação com serviços Google é feita sempre através de HTTPS com encriptação TLS
  • As credenciais de acesso ao Google são geridas exclusivamente pelo sistema de autenticação OAuth da Google — a FaturaApp nunca tem acesso direto às passwords do utilizador
  • Não existe base de dados central da FaturaApp onde os dados dos utilizadores sejam armazenados

⚖️Direitos do utilizador (RGPD)

Nos termos do RGPD, o utilizador tem os seguintes direitos relativamente aos seus dados:

  • Acesso: consultar os dados pessoais tratados
  • Retificação: corrigir dados incorretos ou incompletos
  • Eliminação: solicitar a eliminação dos dados ("direito ao esquecimento")
  • Portabilidade: exportar os dados em formato CSV através da funcionalidade de exportação da aplicação
  • Oposição: opor-se ao tratamento dos dados em determinadas circunstâncias
  • Limitação: solicitar a limitação do tratamento dos dados

Como os dados são armazenados localmente no dispositivo do utilizador, a maioria dos direitos pode ser exercida diretamente através da aplicação ou eliminando os dados do dispositivo.

Para questões relativas a dados sincronizados com serviços Google, o utilizador deve também contactar a Google diretamente.

👶Menores de idade

A FaturaApp é destinada exclusivamente a uso empresarial e profissional, para maiores de 18 anos. A aplicação não recolhe conscientemente dados de menores de 16 anos. Se tomar conhecimento de que um menor forneceu dados, estes serão eliminados imediatamente após notificação.

🔄Alterações a esta política

Esta política de privacidade pode ser atualizada periodicamente para refletir alterações na aplicação ou na legislação aplicável. As alterações serão comunicadas através de:

  • Atualização da data "Última atualização" no topo desta página
  • Notificação na aplicação aquando de alterações significativas
  • Publicação da versão atualizada nesta página

O uso continuado da aplicação após a publicação de alterações constitui aceitação das mesmas.
